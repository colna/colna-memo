## 工作日志 — 04:51 修复 colna sync 顺序 + autostash

- **做了什么**:改 `colna-memo/src/gitsync.rs` 的 `run_sync`:调整为「先 add+commit memory/ → 再 git pull --rebase → reindex 一次 → push」,并给 pull 加 `--autostash`。
- **为什么**:原顺序 pull 在 commit 之前,工作区有未提交改动时 `git pull --rebase` 会被拒(error: cannot pull with rebase: You have unstaged changes),导致 pull 被静默跳过、多设备无法对齐真源。
- **结果**:`cargo build` 通过。先提交本地再 pull,工作区干净;`--autostash` 进一步兜底任何残留未提交改动(如源码改动),rebase 后自动恢复。reindex 由两次合并为一次(commit+pull 后状态稳定再建索引)。
- **关键命令 / diff**:`gitsync.rs` run_sync 步骤重排;pull 改为 `["pull","--rebase","--autostash"]`。
- **沉淀**:见 `30-Resources/troubleshooting/colna-sync.md`。
